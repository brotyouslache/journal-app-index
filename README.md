# journal-app-index

本仓库用于归档和发布多个独立 HTML 页面。页面内容涵盖日常记录、信息整理及静态展示，不关联特定域名或网站。

## 项目简介

`journal-app-index` 是一个静态 HTML 页面集合仓库。每个页面独立存在，可单独访问或嵌入其他场景使用。仓库本身不提供后端服务，不依赖数据库，仅作为页面文件的存储与索引。

## 目录说明

```
.
├── index.html          # 主索引页面
├── pages/              # 独立 HTML 页面存放目录
└── assets/             # 样式、脚本、图片等静态资源
```

- `index.html`：仓库入口页面，列出所有归档页面链接。
- `pages/`：每个 HTML 文件代表一个独立页面，文件名对应页面主题。
- `assets/`：页面引用的公共资源，按类型组织子目录。

## 页面归档说明

所有页面均为完整 HTML 文件，不含外部依赖（除 assets 目录内资源）。页面间无跳转逻辑，各自保持独立。归档内容可能随时间更新，历史版本保留在 Git 提交记录中。

## 维护说明

- 新增页面：在 `pages/` 目录下创建 HTML 文件，并在 `index.html` 中添加链接。
- 修改页面：直接编辑对应文件，提交后更新索引。
- 删除页面：移除文件及索引中的链接，建议在提交信息中注明原因。

本仓库不提供内容审核或版本回滚服务，请自行通过 Git 历史管理变更。

## 使用方式

1. 克隆或下载仓库。
2. 在浏览器中打开 `index.html` 查看页面列表。
3. 点击链接跳转至对应页面，或直接访问 `pages/` 下的文件。
